3. Overview of changes

HTML

Most of what was planned in P2 has been added to the page delivered in P3. The page with the most drastic change from P2 is the guide page. The change was made to make the page more readable, as the page containes alot of text, but solution for how to show the text turned out to workd poorly on mobile. Changes will be made to guide.html in P4.

Styling

The styling of the page was not sepcifiend in much detail in P2, but some of the colors was mentioned and what fonts would be used. The picutes in P2 communicated how the pages should be styled. In the end we have used the colors metioned in P2, but the font-family was changed to Verdana, Geneva, Tahoma or sans-serif.

Javascript

The javascript functionality described in P2 has mostly been used in P3, but some new features have been added and some removed. The JS game was scraped due to time constraint. The navbar and the footer was made into custom HTML elements in JS files. This was not planned in P2, but was done in P3 to not reuse code.

4. Specific changes and reasoning

guide.html

Made divs for all steps and added corresponding text. Made corresponding divs that served as buttons with onclick functionality used in guide.js. Placed buttons and the text content in different divs to make styling easier. These were also placed into a main div to avoid having to style the body.

guide.js

Decided to instead of slideshow functionality, use a more simplistic button functionality. The buttons now hide all paragraphs other than the one that's linked to that button. This as it felt more fitting and easier to navigate from one end to the other. This was harder with using a slideshow as there were 9 steps.

guide.css

Styled with a height so that scrolling isnâ€™t possible. Made the buttons round, as to be more visually pleasing and Changed colors to match the color scheme. Added different settings when screen width is below 800 pixels. This to prevent unnecessary whitespace and bad sizing on mobile devices.

info.html

My original plan was using a grid as a display form for the website. But after further consideration I figured it was better with flex. I really like the way the scroll-snap setting looks so I wanted to imply that in my webpage. Since my background is pictures I decided I would not need more img on my page. Therefore I made buttons to link to the two separate web pages I needed to link. On the buttons I use css to make them appear as buttons you could press.

info.js

I also took inspiration from W3Schools for the js of the slideshow. The js was simple and easy to understand and link to the html.

info.css

As I stated earlier I really liked the way snap-scroll works on pages. Therefore the css to the webpage is centered around this ability. I wanted the information on the page to be in the middle with bright white color. I dimmed the photos used as background so they would not take too much attention from the user.

SlideShow.html

For the slideshow I took inspiration from W3Schools. I used a simple and structured html with plenty of divs for the different img.

Slideshow.js

I also took inspiration from W3Schools for the js of the slideshow. The js was simple and easy to understand and link to the html.

Slideshow.css

I wanted the slideshow to be the main part of the page. And therefore did not add other elements to the webpage. Except for the navbar, I wanted the slideshow to have 100vh so you could not scroll on the page. The placement of the navbar will still be on the top of the page over the photo. This will make it easy for the user to navigate through the page even when enjoying the slideshow.

challenges.html

The making of the challenges page followed the description from P2. The page consists of scrollable cards with 21 challenges with every card having following the purple theme of the website.

challenges.js

challenges.css

hovedside.html

The homepage ended up being similar to how it was described in P2. It is a scrollable page, using scroll-snap functionality. This makes the page mobile-friendly. An animation was added, which was not the plan, but it communicates the scrolling functionality to the user.

hovedside.css

The styling of the navbar for large screens is very similar to how the navbar was described in P2. It was styled with a mobile first approach. On small screens the navbar fills the screen, goes vertical and hides the logo in the middle of the navbar. The mobile logo and the hamburger menu icon is shown. The CSS file is not used in production, but the code is in navbar.js.

navbar.html

The navbar was made with a mobile first approach. The nav menu is vertical on small screens and accessed by a hamburger menu icon. The HTML file is not used in production, because a choice was made to make the navbar a JS component, to not reuse code on every page. But the HTML file was used to develop the scope of the navbar.

navbar.js

The navbar JS file makes a component out of the navbar. The navbar is used on the pages by importing the script and using the custom HTML tag . This was not planned in P2, but we found this was the best solution to not reuse code. The script also closed the vertical navbar when the screen-size becomes larger than 1000px. This is to fix a bug where the vertical navbar would stay open, when going to desktop size, which would also affect styling. Both styling and the HTML of the navbar is found in navbar.js.

Verv.html

Nothing changed

Verv.js

All the cards had an interactive component added to it. It was to make the site more interactive. When the cards are clicked it reveals the content (information about the voluntary work) that was hidden before. It also applies a visual indicator that the card has been clicked.

Verv.css

A blur effect was added to the cards, when a mouse hovers over it, while it is disabled during mobile use. All the cards got a picture corresponding to the position.
